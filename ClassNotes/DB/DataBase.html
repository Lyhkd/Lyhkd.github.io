    Chap1 Introduction¶

    Database-System Applications¶

    数据库管理系统(DataBase-ManagementSystem,DBMS): 由一个互相关联的数据的集合和一组用以访同这些数据的程序组成。

    这个数据集合通常称作数据库(database)，其中包含了关于某个企业的信息。

    DBMS的主要目标是要提供一种可以方便、高效地存取数据库俏息的途径。

    数据库系统的目标¶

    文件处理系统(file-proceesing system)具有一些缺点，而数据库系统可以解决这些问题。

    This typical file-processing system is supported by a conventional operating system.

    The system stores permanent records in various files, and it needs different application programs to extract records from, and add records to, the appropriate files.

    1. Data redundancy and inconsistency：同一个数据可能在多个地方被存储，当某一个地方的数据被更新时，其他数据不能被同时更新。这样就会导致数据不一致。
    2. Difficulty in accessing data：需要某些特定数据，如居住在某个地区的所有学生时，原始设计者可能未提供相应的程序，所以要么就要手动处理数据，要么就要重写程序。这会导致每次想获得一些数据都十分困难。
    3. Data isolation. 数据分散在各种不同格式的文件中。
    4. Integrity problems 数据库中所存储数据的值必须满足某些特定的一致性约束 (consistency constraint)。然而，当新的约束加入时，很难通过修改程序来体现这些新的约束。
    5. Atomicity problems. “atomatic”是计算机科学中的一个概念，它指的是一组操作要么全部执行成功，要么全部不执行，不会出现部分成功、部分失败的情况。如果一个操作执行到一半故障了，就会造成数据库状态的inconsistency。
    6. Concurrent-access anomalies.（并发访问问题）多个用户同时改变数据时，会导致数据的不一致。
    7. Security problems.并非所有用户都有权限访问所有权限

    Data Models¶

    定义

    a collection of conceptual tools for describing data, data relationships, data semantics, and consistency constraints.

    1. Relational Data Model：使用表格的集合来表示数据和数据间的关系。
    2. Entity-Relationship Data Model
    3. Semi-structed Data Model：允许那些相同类型的数据项含有不同的属性集的数据定义。这和早先提到的数据模型形成了对比:在那些数据模型中所有某种特定类型的数据项必须有相同的属性集。（不太懂）可扩展标记语言(eXtensible Markup Language，XML.)被广泛地用来表示半结构化数据
    4. Object-Based Data Model

    关系数据模型¶

    Data Abstraction(数据抽象)¶

    • Physical level：描述数据是怎样存储的。最底层的数据结构。比如一串数据可能是连续存储在某个存储块上的。

    • Logical level：描述存储了什么数据，以及数据间的关系。数据库管理员一般在这个层次工作。

    物理数据独立性(physical data independence)：虽然逻辑层的简单结构的实现可能涉及复杂的物理层结构，但理辑层的用户不必知道这样的复杂性。也就是说，改变物理层的数据结构不会对逻辑层造成影响。

    • View Level：最高层次的抽象，只描述整个数据库的某个部分。系统可以为一个数据库提供多个视图。（类似于Notion中选择部分数据展示，可以以表格、画廊、日历等等形式进行展示）

    Instances and Schemas (实例和模式)¶

    definition

    The collection of information stored in the database at a particular moment is called an instance of the database. 特定时刻存储在数据库中的信息的集合称作数据库的一个实例(instance)。
    The overall design of the database is called the database schema. 数据库的总体设计称作数据库模式(schema)

    根据抽象层次，数据库系统可以分成多个模式。物理模式 (physical schema)在物理层描述数据库的设计，而逻辑模式(logical schema)则在逻辑层描述数据库的设计。数据库在视图层也可以有几种模式，有时称为子模式(subschema)，它描述了数据库的不同视图

    Database languages¶

    Abstract

    A database system provides a data-definition language (DDL) to specify the database schema and a data-manipulation language (DML) to express database queries and updates.
